# SuperHome Design DNA

## Golden Rule

**If it feels like work, it's wrong. Redesign it.**

## Core Principles

### 1. One Thing Per Screen
- Every screen does ONE thing
- Name? One screen. Email? One screen. Address? One screen.
- Never combine inputs. Never overwhelm.

### 2. Zero Decision Fatigue
- User should never have to think
- AI decides. User confirms.
- Max 3 options per choice (never more)
- Default the best option. Let them override if they want.

### 3. Instant Gratification
- Every tap must feel like progress
- Show immediate feedback
- Animations are fast (200-300ms max)
- Loading states are beautiful, never blank screens

### 4. Talk, Don't Type
- AI chat is the primary interface
- Natural language over forms
- If we must use forms, one field at a time
- Auto-fill everything possible

### 5. No Dead Ends
- Every screen has a clear next action
- Back button always works
- Error states offer solutions, not just messages
- Empty states guide the user forward

### 6. Visual Hierarchy
- One primary action per screen (big, bold, obvious)
- Secondary actions are subtle
- Use whitespace aggressively
- Large touch targets (min 48px)

### 7. Colors & Typography
- Clean, minimal palette
- Primary action color: bold and unmistakable
- Text: max 2 font weights per screen (regular + bold)
- Large readable text, never tiny

### 8. The 3-Second Rule
- User should understand any screen in 3 seconds
- If it needs explanation, simplify it
- No tutorials. No onboarding carousels. The app IS the tutorial.

## Forbidden Patterns

- ❌ Long forms with multiple fields
- ❌ Dropdown menus (use tappable options instead)
- ❌ Tiny text or buttons
- ❌ More than 3 choices at once
- ❌ Modals on top of modals
- ❌ Required scrolling to find the main action
- ❌ Loading spinners without context
- ❌ Error messages without solutions
- ❌ Onboarding tutorials or walkthroughs

## Required Patterns

- ✅ Full-screen inputs (one per screen)
- ✅ Bottom-anchored primary CTA
- ✅ Progress indicators for multi-step flows
- ✅ Haptic feedback on actions (mobile)
- ✅ Smooth transitions between screens
- ✅ Skeleton loading states
- ✅ Pull-to-refresh where applicable
- ✅ Swipe gestures for common actions
